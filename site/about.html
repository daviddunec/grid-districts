About the project

Built in the open, verified to the last person

Grid Districts is an open-source project with one goal: give the country a congressional-district baseline that requires zero trust in any person or party — and make it easy enough that anyone can check it.

Who made this

Grid Districts was created by Mark Dunec, CRE, MAI, FRICS and David Dunec. It is not affiliated with any party, campaign, or government body, and it favors no one: the algorithm never sees race, party, or any person’s identity.

The data

  • 2020 Census PL 94-171 block counts, placed with TIGER/Line block coordinates: the official population of every census block in America — the algorithm’s only modern input.
  • Census cartographic state boundaries for grids and the maps on this site.
  • Official decennial county populations, 1950–2010, for the historical demonstration. Complete digital block-level data only exists from 1990 onward, so for consistency the historical demonstration uses official county totals for every decade, scaling today’s street-level settlement pattern to match — a labeled estimate that demonstrates the process, not a literal reconstruction.
  • Census Bureau Vintage 2025 population estimates (July 1, 2025) for the “since the census” figures and the 2030 seat preview. The preview uses the official Huntington–Hill apportionment method, and our implementation had to reproduce the real 2020 apportionment of all 435 seats exactly before it was allowed to touch the estimates.

Coverage, defined: the share of that decade’s official county population records we located and matched. Across the 321 multi-district historical runs, the median is 100.0% and the lowest is 87.46% (Florida’s earliest decades); the gap is scaled, not guessed, and every map shows its own number.

How it’s verified

Every state must pass five gates, in order:

  • Exact head-count match at grid build — the grid’s total must equal the state’s official resident population to the person. A single missing person fails the build.
  • Coverage — every in-state square lands in exactly one district.
  • Contiguity — every district is one connected piece.
  • Deviation gate — the worst district within 2% of an equal share, with dense-state exceptions flagged rather than hidden.
  • Determinism — re-runs, including deliberately shuffled input, must reproduce the map byte-for-byte under cryptographic hash (a short digital fingerprint that changes if even one byte changes).

Nationally, the assigned total across all 435 districts equals the official 50-state resident population — 330,759,736 — exactly.

Beyond the gates, the project ran adversarial review on itself: independent verification code written against a published data contract, blind multi-reviewer evaluation of candidate methods, a hostile review of the report that initially ruled it not publishable (the problems it found were fixed, and the episode is recorded in the public failure log as FL-012), and a public defect log — 12 issues so far, each with root cause and fix. This website itself went through the same treatment: a four-reviewer adversarial pass on its facts, clarity, accessibility, and code. No external expert has reviewed the work yet; obtaining that review (from GAO or the Census Bureau) is the report’s explicit ask.

Run it yourself

The repository — github.com/daviddunec/grid-districts — contains the full engine, data pipeline, verification suite, this website’s generator, and the report. With Node.js installed:

git clone https://github.com/daviddunec/grid-districts.git && cd grid-districts && npm install
node cli.js all --state CO     # grid → districts → verify → render, one state
node scripts/run-all-states.js # the full 50-state production run
node scripts/site-export-data.js && node scripts/site-build.js   # rebuild this website

The code also checks once a year for newly published census data, so each new decade’s baseline can be rebuilt and re-verified automatically.

The fine print

MIT license. The historical maps are labeled estimates. The 2020 maps are a published benchmark — enacted maps must additionally satisfy near-zero deviation and the Voting Rights Act, which is why the proposal is a disclosure standard, not a replacement mapmaker.
